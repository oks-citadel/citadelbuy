## 4. Frontend Animation Blueprint

### 4.1 Performance Constraints

All animations MUST:
- Use CSS transforms (translate, scale, rotate, opacity)
- Be GPU-accelerated
- Not affect layout (no width/height animations)
- Respect `prefers-reduced-motion`
- Load progressively

#### Scroll-Triggered Animations
```typescript
// Use IntersectionObserver for scroll animations
const useScrollAnimation = (threshold = 0.1) => {
  const ref = useRef<HTMLDivElement>(null);
  const [isVisible, setIsVisible] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => setIsVisible(entry.isIntersecting),
      { threshold }
    );
    if (ref.current) observer.observe(ref.current);
    return () => observer.disconnect();
  }, [threshold]);

  return { ref, isVisible };
};
```

### 4.4 Low-End Device Fallbacks

```typescript
// Detect low-end devices
const isLowEndDevice = () => {
  if (typeof navigator === 'undefined') return false;

  const memory = (navigator as any).deviceMemory;
  const cores = navigator.hardwareConcurrency;

  return (memory && memory < 4) || (cores && cores < 4);
};

// Reduce animations for low-end devices
const useReducedAnimations = () => {
  const [reduced, setReduced] = useState(false);

  useEffect(() => {
    const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)');
    setReduced(mediaQuery.matches || isLowEndDevice());
  }, []);

  return reduced;
};
```

## 5. Integration Strategy

### 5.1 Zero-Conflict Principles

1. **Additive Only** - New features add, never modify existing APIs
2. **Feature Flags** - All new features behind flags
3. **Namespacing** - New modules in dedicated directories
4. **Backward Compatible** - Existing contracts preserved
5. **Idempotent Migrations** - Database migrations re-runnable

### 5.2 New Module Structure
