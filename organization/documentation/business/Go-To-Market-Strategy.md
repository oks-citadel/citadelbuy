### 6.3 Feature Prioritization Framework

**RICE Scoring**
```
Score = (Reach Ã— Impact Ã— Confidence) / Effort

Reach: How many users will benefit? (per quarter)
Impact: How much will it improve experience? (0.25 - 3)
Confidence: How sure are we? (50% - 100%)
Effort: How many person-months? (0.5 - 20)
```

**Example Prioritization**

| Feature | Reach | Impact | Confidence | Effort | RICE Score | Priority |

| Live Shopping | 200k | 3.0 | 70% | 8 | 53 | Medium |
| VR Showroom | 50k | 2.5 | 50% | 12 | 5 | Low |

### 6.4 Technical Debt Management

**25% Rule**
- Allocate 25% of engineering capacity to tech debt
- Every sprint includes refactoring
- No shipping features on unstable foundation

**Debt Categories**
1. **Critical** (Fix immediately)
   - Security vulnerabilities
   - Data loss risks
   - System instability

2. **High** (Fix within sprint)
   - Performance issues
   - Scalability problems
   - User-facing bugs

3. **Medium** (Fix within quarter)
   - Code quality issues
   - Test coverage gaps
   - Documentation

4. **Low** (Fix when convenient)
   - Code style issues
   - Minor refactors
   - Nice-to-haves
